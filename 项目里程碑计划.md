# PocketCoder 项目里程碑计划

## 总体说明
- 本计划用于把 [项目完成计划.md](D:/develop/study/AI/PocketCoder/项目完成计划.md) 拆分为可执行的里程碑。
- 里程碑按依赖顺序推进，默认不并行跳步。
- 每个里程碑都以“可验证交付物 + 明确退出条件”为完成标准，而不是只完成代码编写。
- 总目标固定为跑通 V1 主链路：桌面启动 `codex` 会话、手机完成配对、手机查看实时输出、手机发送 prompt、手机处理授权、桌面恢复本地输入后远控立即失效。

## Milestone 1. Monorepo 基础与工程基线
### 目标
- 建立统一的 TypeScript monorepo 和 4 个 workspace 基础结构。

### 交付物
- 根目录 `package.json`、workspace 配置、`tsconfig.base.json`、`eslint.config.mjs`、`.gitignore`。
- 4 个固定 workspace：`apps/web`、`apps/agentd`、`apps/relay`、`packages/protocol`。
- 根脚本最小集合：`dev`、`build`、`lint`、`typecheck`、`test`。

### 退出条件
- 仓库结构与 [V1-项目结构.md](D:/develop/study/AI/PocketCoder/设计方案/V1-项目结构.md) 一致。
- 根层可以统一执行安装、构建、lint 和测试入口。
- 业务代码、部署文件、文档与 skill 资源边界清晰，没有相互混放。

## Milestone 2. 协议层完成
### 目标
- 完成 `packages/protocol`，形成唯一共享契约源。

### 交付物
- 统一 envelope、消息类型、会话状态枚举、错误码、Zod schema。
- 配对、设备注册、会话订阅、输出增量、授权、prompt、中断、恢复桌面控制等首版协议定义。

### 退出条件
- `relay`、`agentd`、`web` 后续不再重复定义共享协议结构。
- 协议 schema、类型导出、兼容性测试骨架齐备。
- 所有跨端消息都可通过协议包统一校验。

## Milestone 3. Relay 基础服务完成
### 目标
- 完成 `apps/relay` 的最小可运行服务骨架。

### 交付物
- `transport/http`、`transport/ws`、`security`、`storage`、`modules`、`infra` 分层。
- SQLite 元数据存储边界。
- 配对入口、健康检查、WebSocket 连接、订阅、心跳、回放、去重骨架。

### 退出条件
- Relay 能启动并暴露最小 HTTP/WS 服务。
- Relay 只处理设备注册、路由、密文转发和回放，不承载 AI 会话逻辑。
- SQLite 能持有设备、token、路由、心跳、加密事件和去重记录。

## Milestone 4. Agentd 桌面执行骨架完成
### 目标
- 完成 `apps/agentd`，使桌面端成为本地事实来源。

### 交付物
- `cli`、`providers/codex`、`sessions`、`transport`、`security`、`platform/windows`、`infra` 分层。
- 真实 `codex` PTY 启动能力。
- 本地会话注册表、状态机、事件发布和命令处理骨架。

### 退出条件
- `agentd` 能启动真实 `codex` 会话。
- 会话输出可被解析为结构化事件。
- 可接受 `prompt`、`approval`、`interrupt`、`resumeDesktopControl` 等命令入口。

## Milestone 5. Web PWA 主界面完成
### 目标
- 完成 `apps/web` 的移动端优先 PWA 页面壳和浏览器侧边界。

### 交付物
- 固定页面：`/pair`、`/sessions`、`/sessions/[sessionId]`、`/connection-error`。
- `features/`、`lib/crypto/`、`lib/realtime/`、`lib/storage/`、`lib/pwa/` 边界。
- 主链路 UI 壳：配对入口、会话列表、实时输出区、底部 prompt 输入、授权卡片、释放控制权操作。

### 退出条件
- 关键页面都可访问。
- 页面层只做装配和交互，不直接承载深层协议、实时或密码学实现。
- PWA 基础能力可用，页面结构满足移动端优先要求。

## Milestone 6. E2EE 配对与设备注册完成
### 目标
- 打通三端配对、安全注册和消息级 E2EE 接入边界。

### 交付物
- `agentd auth` 一次性 token 生成。
- 桌面端和浏览器端长期密钥生成与存储边界。
- token 发起配对、公钥交换、设备注册、访问能力发放。
- relay 仅持有最小元数据和密文，不持有明文。

### 退出条件
- 有效 token 可以完成配对。
- 过期 token、已使用 token、撤销设备被拒绝。
- 未配对设备不能订阅会话。
- 错误设备密钥不能读取历史。

## Milestone 7. 主链路联调完成
### 目标
- 跑通手机与桌面之间的实时控制主链路。

### 交付物
- `agentd` 会话输出实时推送到 `web`。
- `web` 发送 prompt，桌面端只执行一次。
- 授权请求展示和 allow/deny 回传闭环。
- 手机端断线重连后的最近窗口回放。

### 退出条件
- 从桌面启动会话到手机侧查看输出全链路可用。
- prompt、授权、订阅、重连、回放行为符合协议预期。
- `messageId` 去重生效，不重复执行命令。

## Milestone 8. Windows 控制恢复与远控失效完成
### 目标
- 完成 Windows 本地输入恢复检测和远程控制失效闭环。

### 交付物
- `apps/agentd/src/platform/windows/` 下的输入检测、控制状态、平台接线。
- 显式“把控制权还给桌面端”与本地键盘恢复输入的统一语义。
- 向会话层和事件层传播远控失效状态。

### 退出条件
- 桌面端一旦恢复本地输入，当前会话远控立即失效。
- 手机端可以收到状态变化。
- 后续远程控制命令会被阻断或拒绝。

## Milestone 9. 自托管部署完成
### 目标
- 完成单机自托管的 `relay + SQLite` 部署骨架。

### 交付物
- `compose.yaml`
- `.env.example`
- `infra/docker/relay.Dockerfile`
- 数据卷、环境变量、健康检查、重启策略

### 退出条件
- `docker compose config` 可通过。
- 单机环境可以启动 relay。
- SQLite 数据目录、环境变量和容器运行参数符合设计约束。

## Milestone 10. 测试与验收完成
### 目标
- 补齐自动化测试和主链路验收，形成 V1 收口结果。

### 交付物
- `protocol` 的 schema/兼容性测试。
- `relay` 的单测和 WebSocket/回放/去重集成测试。
- `agentd` 的状态机/事件流/幂等/本地控制恢复测试。
- `web` 的单测和冒烟测试。
- 一份最终验收记录，明确主链路已验证项和未实现项。

### 退出条件
- `lint`、`typecheck`、相关测试通过。
- 高风险项都有回归覆盖：配对、加密、权限、幂等、中断、重连、状态机。
- 主链路端到端可在 Windows 开发机和单机自托管环境中跑通。

## 建议执行顺序
- 第 1 步：Milestone 1 到 Milestone 2
- 第 2 步：Milestone 3 到 Milestone 4
- 第 3 步：Milestone 5 到 Milestone 6
- 第 4 步：Milestone 7 到 Milestone 8
- 第 5 步：Milestone 9 到 Milestone 10

## 默认约束
- 默认只做 V1，不提前扩展多用户、多租户、跨平台桌面和云部署。
- 默认桌面端仅支持 Windows。
- 默认移动端仅支持 Next.js PWA，不开发原生 App。
- 默认优先保证主链路可用和可验证，再补充细节和增强能力。
